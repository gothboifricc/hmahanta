Web Scraping - What is it? 🤔

Web scraping is a process or a method to automate gathering information (hence the word 'scraping' that comes from the word 'scrape' or 'to scrape something off') from websites.

Imagine you want to collect all the prices of your favorite video games from different online sites. Instead of visiting each site and writing down the prices one by one, web scraping lets us write a small computer program (generally, python is used as the code language) that does exactly this in just a few seconds.

Tools Used

Scrapy, Selenium, and BeautifulSoup (BS4) are tools and libraries used for web scraping. The choice between which one to use depends on our use case, as each tool has its strengths and weaknesses:

Ideal Use Cases

🔹Scrapy:

Scraping large volumes of data from multiple pages quickly, especially when pages are structured similarly and don’t rely heavily on JavaScript.


🔹Selenium:

Web scraping for sites that rely on JavaScript and dynamic loading, or when you need to interact with complex forms and elements like buttons, dropdowns, and other interactive elements.


🔹BeautifulSoup (BS4):

Great for smaller projects where you don’t need the complexities of Scrapy or Selenium. it can parse and search the HTML content quickly and easily (Parse means to break down the code inside the site into smaller components in order to make it easily understandable so that it can be analysed with ease.)


Simply put, for most large, static sites, Scrapy is generally the best choice due to its speed and built-in scraping utilities. For dynamic, interactive sites, that are dependent on JavaScript, Selenium is preferable. If you only need to parse HTML for a few pages/small-scale scraping tasks, BeautifulSoup can be the most straightforward option.
