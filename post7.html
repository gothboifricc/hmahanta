Aggregate Functions vs. Window Functions

SQL • September 24, 2024
Scroll

Functions in SQL

In SQL, functions are powerful tools that helps perform various tasks on our data directly within the queries. Functions allow us to transform, summarise (aggregate) and manipulate data to get insights or prepare it for analysis. They can be used for simple tasks, like rounding numbers, to more complex ones, like calculating averages or filtering text.


Today, I would like to dive into two function types, namely, Aggregate Function and Window Function.


Aggregate Functions

These type of functions are used to perform calculations on multiple rows of a table and return a single value. Often used with the GROUP BY clause to group rows that share a common attribute.

Example

ðŸ”¹The SUM() function:

SELECT Product, SUM(SalesAmount) AS TotalSales
FROM SALES
GROUP BY Product;

Window Functions

These are used to perform calculations across rows of data related to the current row, without collapsing them into a single summary.

These are indicated by the 'OVER' clause.

Example

ðŸ”¹RANK(), DENSE_RANK(), ROW_NUMBER(), SUM() etc.

Basic structure of a window function looks something like this:

--- window_function() OVER (PARTITION BY column1 ORDER BY column2)

It is important to note that 'SUM()', 'AVG()', 'COUNT()', 'MIN()', 'MAX()' etc. are all examples of aggregate functions but can be used as window functions when combined with the OVER() clause.
