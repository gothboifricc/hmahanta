'=' and 'IN' keywords in Nested Queries

SQL • September 7, 2024
Scroll

Introduction

Consider this query for a second:

SELECT client_.client_name FROM client_
WHERE client_.branch_id IN (
SELECT branch.branch_id FROM branch
WHERE branch.mgr_id = 102
);

And now, this query:

SELECT client_.client_name FROM client_
WHERE client_.branch_id IN (
SELECT branch.branch_id FROM branch
WHERE branch.mgr_id = 102
LIMIT 1
);


Apart from the 'LIMIT' clause, there does not seem to be any apparent difference between the two queries, right? Wrong.

Let us understand what

The '=' operator in the first query is used for equality comparisons. It means you should use '=' when you expect a single value as the result of a query. Within the context of the latter query above, it fetches client names where branch_id is exactly 102, i.e. it expects branch_id to match a single specific value.

On the other hand, the 'IN' keyword in the second query is used to compare a value against a list of values. It means you should use IN when you want to check if a value matches any value in a set or list. It can be a static list or a dynamic list returned by a nested query.
