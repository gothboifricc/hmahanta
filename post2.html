What are modules?

Modules in Python are files that contain Python code (like functions, classes and variables) which can be imported and reused in other programs.

Difference between 'import' & 'from import'

ğŸ”¹import my_module: This piece of code imports the module as a whole, meaning we need to reference the module name each time we use something from it.

Example -

import math
print(math.sqrt(16)) #note how we need to prefix the sqrt() function imported from the math module with 'math.'

ğŸ”¹from import: This imports all contents of the module directly, so we don't need to prefix items with the module name.

Example -

from math import *
print(sqrt(16)) #in contrast to previous example, note there is no need to reference the 'sqrt()' function here

Difference in use cases

1. Use 'import my_module' when we want to maintain clarity and avoid conflicts.

Use 'from my_module import *' with caution, especially in larger programs where name conflicts might occur if multiple modules have functions or variables with the same name. Also, keep in mind when we use this, it is harder to track where functions or variables come from since they are not prefixed with the module name.
