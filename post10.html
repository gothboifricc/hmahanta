Delimiters: What it is, Purpose & Use

SQL • September 12, 2024
Scroll

What are Delimiters?

This is one of the topics that someone can rightly consider as an intermediate-advanced level concept to wrap our heads around. However, let us try to grasp the idea behind it.

Delimiters are special keywords (symbols/characters) that mark the beginning and end of blocks of code or various elements in SQL statements. Some examples of delimiters include the default semicolon ';', comma ',', backtick '`', or custom delimiters like '$$' etc.

Purpose & Use

Delimiters are sometimes temporarily changed to inform SQL to wait until it encounters the new delimiter before it treats the new block of code as complete.

Triggers (automated actions executed in response to specific events), or other complex scripts in SQL can contain many statements, each needing a semicolon to terminate it properly. Here, the default semicolon (;) delimiter can cause issues because semicolons are used to terminate individual SQL statements. So, if SQL interprets a semicolon inside a trigger as the end of the entire trigger or script, it will return an error. Changing the delimiter in cases like these allows the semicolons inside the block to be used normally to terminate individual SQL statements without any confusion.

Conclusion

Simply put, delimiters are symbols that separate and organise elements within a query, ensuring that SQL correctly interprets and executes the code. They help keep the query structured and easy to work with.

Happy Learning!
