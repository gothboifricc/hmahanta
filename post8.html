Cardinality of Relationships in Data Modeling

Data Modeling • September 20, 2024
Scroll

What is Cardinality?

In the context of databases and Power BI, cardinality refers to the uniqueness of data values contained in a column. It describes the relationship between tables and the type of connection that exists between them.

To be precise, it defines how the rows in one table (say Table A) are related to the rows in another table (say Table B) based on a specific column or columns.


Main types of Cardinality

1. One-to-One (1:1):

Each record in Table A is linked to one and only one record in Table B, and vice versa.

Example: Each employee has one unique employee ID, and each employee ID corresponds to only one employee.

2. One-to-Many (1:M):

One record in Table A can be related to multiple records in Table B, but each record in Table B is linked to only one record in Table A.

Example: One customer can have multiple orders, but each order is linked to only one customer.

3. Many-to-One (M:1):

This is just the reverse of One-to-Many. Multiple records in Table A can relate to one record in Table B.

Example: Multiple employees may belong to one department, but each department can have many employees.

4. Many-to-Many (M:M):

Records in Table A can relate to multiple records in Table B, and vice versa.

Example: Students can enroll in multiple courses, and each course can have multiple students.

Proper understanding of cardinality (like one-to-many or many-to-many) enables Power BI to deliver accurate insights, optimise performance, and handle complex data models without redundancy or calculation errors.
